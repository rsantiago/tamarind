# Semantic CSS

If you look at the `style.css` generated by Tamarind, you might notice something strange. There are very few classes.

Instead, we style the elements themselves.

```css
h1 {
    font-family: 'Playfair Display', serif;
    font-size: 3rem;
}

p {
    line-height: 1.6;
    margin-bottom: 1.5rem;
}
```

## Why?

Tailwind and Bootstrap encourage "Utility First" CSS. This is great for apps, but for documents (like blogs), it creates noise. `class="text-xl font-bold mb-4 text-gray-800"` obfuscates the content.

By styling the tags directly (`article h2`), we keep the HTML clean. The writer doesn't need to know about CSS classes. They just write Markdown headers, and the global stylesheet takes care of the rest.

## Dark Mode

We use CSS variables to handle theming:

```css
:root {
    --bg-color: #ffffff;
    --text-color: #111111;
}

[data-theme="dark"] {
    --bg-color: #111111;
    --text-color: #ffffff;
}
```

A simple Javascript snippet toggles the `data-theme` attribute on the `html` tag, and the browser repaints the page instantly.
