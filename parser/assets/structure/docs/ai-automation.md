# The "Standard Output" for AI

We are entering an era where content is increasingly co-authored or fully generated by Artificial Intelligence. Most web frameworks (React, Vue, Next.js) are complex, fragile, and "hallucination-prone" when coded by LLMs.

**Tamarind is different.** It was designed to be the perfect target for AI output.

## Why Tamarind is AI-Friendly

1.  **Markdown Native**: LLMs speak Markdown fluently. Tamarind consumes it directly. No complex JSON schemas or CMS APIs to hallucinate.
2.  **Deterministic Shortcodes**: Instead of asking an AI to "code a chart in D3.js" (which often fails), you ask it to "output a Mermaid block". The syntax is strict, simple, and token-efficient.
3.  **Single File Context**: An entire article, including its metadata, styles, and diagrams, sits in a single `.md` file. This fits perfectly into an LLM's context window.

## Prompting for Tamarind

You can turn any LLM (ChatGPT, Claude, Gemini) into an expert Tamarind author by providing a simple "System Instruction".

### Copy-Paste System Prompt

Use this prompt to instruct your AI assistant:

```markdown
You are an expert technical writer using the Tamarind SSG.
Rules for generating content:
1. Output purely in Markdown.
2. Use Front Matter at the top (title, subtitle, date, tags).
3. For diagrams, ALWAYS use the mermaid shortcode: {{{!}}{ mermaid }} ... {{{!}}{ /mermaid }}.
4. For math, use LaTeX inside the math shortcode: {{{!}}{ math }} ... {{{!}}{ /math }} or $$ ... $$.
5. For alerts, use callouts: {{{!}}{ callout type="warn" }} ... {{{!}}{ /callout }}.
6. Keep the tone minimal, clean, and professional.
```

## Automating your Blog

Because Tamarind is a single binary with zero dependencies (`npm` is not required), it is the ideal engine for **Autonomous Agents**.

An agent can:
1.  **Research** a topic online.
2.  **Write** a `.md` file with diagrams and citations.
3.  **Run** `./tamarind build`.
4.  **Deploy** the resulting `./website` folder.

All without human intervention or breaking environment builds.

### Example Agent Workflow

Imagine a Python script or an Agent protocol that runs daily:

```python
# Pseudo-code for an Autonomous Editor
topic = agent.find_trending_topic("System Design")
article_md = agent.write_article(topic, style="tamarind-markdown")

save_file("articles/daily-trend.md", article_md)
subprocess.run(["./tamarind", "build"])
upload_to_s3("./website")
```

Tamarind provides the reliability required for these autonomous pipelines.
