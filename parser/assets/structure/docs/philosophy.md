# Digital Permanence

Tamarind is built on a specific set of beliefs about software and the internet.

## 1. Tools Should Be Invisible

The best tool is the one you forget you are using. If you spend more time configuring your blog than writing for it, the tool has failed. Tamarind aims for zero configuration. The directory structure *is* the configuration.

## 2. Content is King, Context is Queen

Your words matters most. But the way they are presented (typography, spacing, layout) respects the reader. We do not use "engagement hacks", popups, or sticky headers. We provide a calm, clean reading environment.

## 3. Dependency Minimalism

Modern software development has a dependency addiction. `node_modules` folders with 200MB of code for a simple blog is a liability. It creates security risks, maintenance burden, and inevitable "bit rot." Tamarind has **zero** runtime dependencies.

## 4. The 100-Year Web

We generate plain HTML and CSS. This format has worked for 30 years and will likely work for 100 more. By avoiding complex JavaScript frameworks for static content, we ensure your site is readable on everything from a high-end workstation to a ten-year-old smartphone, and eventually, by digital archivists of the future.
